# 🤖 Automation Rules

This milestone focuses on implementing automated SOC workflows using Microsoft Sentinel Automation Rules to improve incident triage, investigation efficiency, incident enrichment, and incident lifecycle management.

The automation workflows were designed using real incidents generated within the lab environment and demonstrate how SOC teams automate repetitive operational tasks to reduce manual effort and improve response efficiency.

The implemented automation rules cover:
- automated incident assignment
- high severity incident enrichment
- automatic low severity incident closure
- SOC workflow standardization
- incident lifecycle automation

These workflows simulate real-world Security Operations Center (SOC) automation practices used to streamline alert handling and reduce analyst workload.

# 📂 Automation Rule Navigation

| Automation Rule | Description |
|---|---|
| [01-Auto-Assign-Incidents.md](01-Auto-Assign-Incidents.md) | Automatically assigns brute-force related incidents to SOC analysts during incident creation |
| [02-High-Severity-Tagging.md](02-High-Severity-Tagging.md) | Automatically enriches and prioritizes high severity incidents using investigation and classification tags |
| [03-Auto-Close-Informational-Incidents.md](03-Auto-Close-Informational-Incidents.md) | Automatically classifies and closes low severity informational incidents generated from expected RDP activity |
